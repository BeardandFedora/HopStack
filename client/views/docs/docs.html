Introduction

"Bearded Robot" is a fully responsive web application built with AngularJS, which the end user interacts with in order to perform a multitude of Simflofy tasks. It use Sass CSS which makes it easy for developers to modify, and allows an easy path to customizing the presentation layer, and as independent as possible from any core features of the Simflofy server.

Bearded Robot was built to consume JSON, and that is all.

Features

  • Responsive Design. Excellent device interaction on desktop, tablet and phone.
  • Multiple Themes. By using a popular style framework in Bearded Robot, we easily allow a large community of developers already familiar with the world's leading UI framework...Bootstrap.
  • Web App. (Single Page Application) Built with AngularJS. Fully AJAX powered and prefers to munch on JSON entirely. We prefer that the user never refresh the entire page needlessly. Faster, smoother, more fun.
  • Runnable Task Application. Developer Mode
  • Internationalization (i18n) and Localization (l10n) support. Add English, Español(Spanish), 日本語(Japanese), 中文(Chinese), Deutsch(German), français(French), Italiano(Italian), Portugal(Portuguese), Русский язык(Russian), 한국어(Korean) for multi-lingual demos, and even add it to the core release for all users.
  • Solid Workflow with Yeoman, scaffolding with Yo, dependencies (plugins) management with Bower, tasks automation with Grunt, local and native cloud deployments on popular PaaS platforms like Heroku, EngineYard, AppFog, Amazon AWS, Rackspace, and more.
  • Realtime Form Validation. Lighting fast data interaction, searching, and making progress.
  • Build better apps with Sass CSS (Scss files). Easy modification with variables (great for teams).
  • Built with multiple theme support for end user extension, and in-app upgrades/theming.
  • Multiple page transitions built in for end user configuration.
  • Reusable widgets and components. Available for end user extension through API.
  • Light weight and super fast.
  • Built with Bootstrap 3.
  • Valid HTML5 and CSS3
  • Built-In Cross Browser and Device Support
  • Cloud deployments
  • Easy to use API for developers

How to Run Bearded Robot

Preparation: You'll need to install Node.js (NPM comes along with it) and node-static (an NPM package)

Steps:

Download Node.js here then install it, and then install node-static with one command line

$ npm install -g node-static

The usage of node-static is simple. Just enter the "dist" directory with command line, and type

static

It'll return something like this: 

serving "." at http://127.0.0.1:8080

Then you can visit Bearded Robot at http://127.0.0.1:8080 with your browser

How to Use It (Sass Version, The Default)

In order to do development, you'll need have the following tools installed:

  • Node.js (NPM comes along with it)
  • Yeoman (if you are using npm 1.2.10 or above, this will also automatically install Bower, Grunt) for modern workflow.
  • Ruby, Sass , Compass for Sass CSS.

    • And you'll need to know how to use them.

    1. Enter "bearded-robot-proto" folder with your command line tool.
    2. Install NPM and Bower packages with NPM and Bower.
    3. Then type grunt server to launch the local dev environment.
    4. Then type grunt build when you are ready to make the build and deploy.
    5. The rest of the docs are in the bearded-robot project wiki.

    All the files for your development stack are under the ./bearded-robot/client/ folder.

    Everything else is just content

    Layouts

    There's three ways to change default layout behvior. (1) Change the main app file (ie - index.html), (2) Change the "main.coffee" file inside "./client/scripts/shared" folder, or (3) both (preferred).

    Both ways are easy. Here is an example...by opening "main.coffee", we find the following code. We change the value to update a layout behavior for a control menu.

    $scope.navcontrol =
    layout: 'wide'          # 'boxed', 'wide'
    menu: 'vertical'        # 'horizontal', 'vertical'
    fixedHeader: true       # true, false
    fixedSidebar: true     # true, false

    For example, if we wanedt to change to "boxed layout", then we would update code like so:

    $scope.navcontrol =
    layout: 'boxed'         # 'boxed', 'wide'
    menu: 'vertical'        # 'horizontal', 'vertical'
    fixedHeader: true       # true, false
    fixedSidebar: true     # true, false

    Done :)

    A bit on CoffeeScript

    How can we use plain JavaScript instead of CoffeeScript?

    Well, that's already been done, and we have a ton of javascript to rely on these days for responsive, multi-device apps. Bearded Robot already has custom Grunt tasks to compile all of the simple, light-weight CoffeeScript to JavaScript automatically. So, if you really want to use straight javascript, it's your questions to answer, and the ones to follow.

    When we run “grunt build", all of the CoffeeScript files will be compiled, one by one, into equivalent JavaScript inside a hidden “.tmp” folder, and theoretically, we could replace the CoffeeScript files with those JS files if one were to prefer to use plain javascript. If you break it, you fix it.
